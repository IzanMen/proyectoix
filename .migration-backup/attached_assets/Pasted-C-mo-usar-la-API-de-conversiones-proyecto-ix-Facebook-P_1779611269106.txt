Enviar eventos mediante la API
Una vez que tengas un token de acceso, elige los eventos que quieres enviar desde el servidor, genera la carga y realiza las solicitudes. La carga incluirá información sobre los eventos y los parámetros que quieres enviar.
Realizar una solicitud POST
Para enviar eventos nuevos, realiza una solicitud POST al perímetro /events de la API desde esta ruta: https://graph.facebook.com/{API_VERSION}/{PIXEL_ID}/events?access_token={TOKEN}.
Cuando realizas una solicitud POST a este perímetro, Facebook crea nuevos eventos del servidor.

Tiempo de transacción del evento
El parámetro event_time representa el tiempo de transacción del evento. El tiempo especificado puede ser anterior al momento en que se envía el evento a Facebook. Si el valor de event_time supera en más de 7 días la fecha actual, se devolverá un error para toda la solicitud y no procesaremos ningún evento. Más información
Deduplicar eventos del píxel y del servidor
Facebook intenta deduplicar los eventos idénticos que se envían mediante el píxel de Meta y la API de conversiones. Para determinar si los eventos son idénticos, usamos su identificador y su nombre. Más información
Crear lotes de eventos
Puedes enviar hasta 1.000 eventos en un lote de datos. Sin embargo, para garantizar un rendimiento óptimo, recomendamos que los eventos se envíen tan pronto como se produzcan. Si uno de los eventos de un lote no es válido, rechazaremos todo el lote. Más información

Paso 3: Supervisar las métricas de los eventos
Cuando hayas enviado suficientes datos a través de la API de conversiones, podrás supervisar tres métricas que indican el estado de tus eventos. Si las supervisas, podrás observar su rendimiento y descubrir cómo puedes mejorar los resultados con las prácticas recomendadas, lo que te permitirá optimizar el rendimiento de los anuncios y reducir el costo por resultado.
Calidad de coincidencias de eventos
Esta métrica indica el nivel de eficacia de la información de los clientes para encontrar coincidencias entre instancias de eventos y una cuenta de Facebook.
Tasa de deduplicación
Esta métrica indica el porcentaje de eventos del píxel y de la API de conversiones que se deduplicaron para obtener un recuento preciso.
Actualización de datos
Esta métrica indica el tiempo transcurrido entre el momento en que ocurre el evento y el momento en que lo recibimos para evaluar la probabilidad de que una persona realice la acción deseada después de ver el anuncio.
Cobertura de eventos de la API de conversiones
El envío de los mismos eventos desde el navegador y el servidor ofrece datos más precisos, para que puedas mejorar la segmentación del público y reducir tu costo por resultado.
Ve al servicio de ayuda para obtener más información sobre cada métrica y sus prácticas recomendadas. Para ver una métrica, ve a su pestaña en la página de detalles del evento.
